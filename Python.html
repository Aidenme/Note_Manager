Variable Assignment

Variables are assigned with "=" also known as the assignment operator.

The variable on the left is assigned to the object or variable on the right.

  • cake = "Tasty!"
  • print(cake)
  • "Tasty!"

You can use the assignment operator multiple times in one line to assign different variables to the same value:

  • cat = dog = "Cute Animal!"
  • print(cat)
  • print(dog)
  • "Cute Animal!"
  • "Cute Animal!"

Python has the unique ability to do multiple variable assignments on one line:

  • kelly, andy, mike = 'Wealthy', 'Healthy', 'Stealthy'
  • print("Kelly is " + kelly)
  • print("Andy is " + andy)
  • print("Mike is " + mike)
  • Kelly is Wealthy
  • Andy is Healthy
  • Mike is Stealthy

You can even use this with a list to break it apart into seperate variables:

  • snakes = ["Cobra", "Viper", "Python"]
  • deadly_snake, scary_snake, nice_snake = snakes
  • print(deadly_snake)
  • print(scary_snake)
  • print(nice_snake)
  • 'Cobra'
  • 'Viper'
  • 'Python'

This can save some lines of code in situations where you want to swap variable values. Normally if you want to swap values you need to use a temporary variable like this:

  • book1 = "Maximum Ride"
  • book2 = "Harry Potter"
  • placeholder = book1
  • book1 = book2
  • book2 = placeholder
  • print(book1 + ", " + book2)
  • Harry Potter, Maximum Ride

If you tried this swap without the placeholder, assigning book2 to book1 would cause you to lose the original value in book1. Since with tuples you can do multiple assignments at once you can swap these values with one line and avoid a placeholder:

  • book1 = "Maximum Ride"
  • book2 = "Harry Potter"
  • book1, book2 = book2, book1
  • print(book1 + ", " + book2)
  • Harry Potter, Maximum Ride

You can combine multiple assignment operators with commas if you want:

  • all_flavors = fav_flavor, worst_flavor = ["Vanilla", "Ass"]
  • print(all_flavors)
  • print(fav_flavor)
  • print(worst_flavor)
  • ['Vanilla', 'Ass']
  • 'Vanilla'
  • 'Ass'

You don't even need to have the same number of values on both sides of the equal sign. * can be used to indicate a variable on the left you want to use to absorb the extra variables on the right:

  • tenlb_weight, twentylb_weight, *dog = 10, 20, 8, 16
  • print("Ten pound weights are " + str(tenlb_weight) + " lbs")
  • print("Twenty pound weights are " + str(twentylb_weight) + " lbs")
  • print("Dogs are either " + str(dog) + " lbs")
  • Ten pound weights are 10 lbs
  • Twenty pound weights are 20 lbs
  • Dogs are either [8, 16] lbs

For efficiency, when a variable 'a' is assigned to a variable 'b' python doesn't create a new space in memory for variable 'a', instead variable 'a' just references the same space in memory as variable 'b'. If variable 'b' is assigned something different Python will then copy the original contents of 'b' into a new place in memory now allocated for variable 'a' to reference.

This behavior is different for lists. Since for a list sometimes only part of the value is changed, Python does not bother allocating new memory for a new copy of the list when only some of its values change. The two variables will continue referencing the same list in memory:

  • pets = ['dog','cat','bird','goldfish']
  • animals = pets
  • pets[1] = 'hamster'
  • print(animals)
  • ['dog', 'hamster', 'bird', 'goldfish']

You can create a copy of a list that has its own place in memory if you use slice notation in the assignment instead of just assigning one variable to another:

  • candies = ["Lemonhead", "Candy Corn", "Mints"]
  • treats = candies[:]
  • candies[1] = "Chocolate"
  • print(treats)
  • ['Lemonhead', 'Candy Corn', 'Mints']

You can do the same thing with the copy method from the copy module:

  • import copy
  • pets = ['dog','cat','bird','goldfish']
  • animals = copy.copy(pets)
  • pets[1] = 'hamster'
  • print(animals)
  • ['dog', 'cat', 'bird', 'goldfish']

Note that when you're working with lists nested in lists you're going to want to use the deepcopy() method of the copy module.

Numbers

Number Data Types:

  • Integers
  • Floats
  • Complex Numbers

Integer and float size is infinite (limited by system memory only)

Integers

Operations with them make integers unless dividing. Then they convert to floating point numbers:

  • 1/8
  • 0.125

You can convert a float to a float with int(). No rounding. This removes everything after the decimal.

  • int(4.9)
  • 4

Floats

Operations with them always return floats. Any operation done on a int that needs it to be a float will automatically convert it to a float, but the function float() will convert.

  • float(4)
  • 4.0

Complex Numbers

Numbers made of a real and imaginary part.

The imaginary part is made up of an integer and standard imaginary number notation "j", which represents the square root of 1:

  • 4

You can create a complex number with a real and imaginary part with this function:

  • complex(8, 4j)
  • (8+4j)

The real and imaginary part of the number are floats. Results are returned in paren to show they represent one value, one complex number.

You can also do math with them like this:

  • 3 + 2j - (4 + 4j)
  • (-1 - 2j)

They never auto convert to another type, but can be converted with these functions:

  • z=(3+5j)
  • z.real
  • 3.0

Or to get the imaginary part:

  • z.imag
  • 5.0

Number Types

Python can work with number types other than base 10. Like octal and hexidecimal numbers. Note python will always use base 10 to print numbers.

Start a number with "0o" and it will be treated as an octal.

  • num = 0o12
  • print(num)
  • 10

Start a number with "0x" and it will be treated as a Hexidecimal

  • num = 0x8ab
  • print(num)
  • 2219

Strings

They are immutable. You must make a new string or assign string operation output to the input string to "change" it.

Strings can be delimited with ' and ". To define a string across multiple lines of code delimit with ''' or """. When you define a string with single-quotes, double quotes will be treated as a character and won't delimit. Both double and single quotes won't delimit when using three quotes to define the string across multiple lines. You can also define a string across multiple lines by ending each line with \.

Although strings are immutable you can easily turn a string into a list of characters with the list() function. You can convert it back with "".join(<some list of characters>).

Operators

+

The concatenation operator. Adds one string on to the end of another. Since "+" is also the addition operator adding a integer to a string doesn't automatically convert the integer to a string and causes an error. Also note this is less efficient than the join() method.

*

The repetition operator. Multiplies a string.

  • 'A' * 10
  • 'AAAAAAAAAA'
in

Determines if a string is in a string and returns a boolean if it is (True) or isn't (False)

%

You can use the modulus operator (or .format() method) to display a variable or object as a string and apply formatting to it.

  • "Good %s my good sir" % "morning"
  • 'Good morning my good sir'

The format is a string containing formating codes on the left of % and the variable or object to modify is on the right.

Formating codes start with % and are defined by the character attached to the right. They also mark where in the string formatted values should appear.

In this case the formatting code is "%s" which means apply str() to the passed values on the right of modulus.

Here is a summary of the formatting codes:

CodeCode Meaning

%Literal percent sign

Between the code and % sign you can also put a number. This determines the size of the field returned values will be placed in:

  • "The number %8s is the loneliest" % 1
  • 'The number        1 is the loneliest'

You can also put .# after this number to determine how many decimal placed should be included in a result when working with floating point numbers:

  • "246 is %.2f percent of 787" % ((246 / 787) * 100)
  • '246 is 31.26 percent of 787'

You can format multiple values at once if you provide a tuple or dictionary:

  • "%s %s you are late for %s!" % ("David", "Clancy", "School")
  • 'David Clancy you are late for School!'

This is much clearer with a dictionary since you can specify the name of the value you want like this:

  • dictionary = {"first_name" : "James", "last_name" : "Kirk", "location" : "The Enterprise"}
  • "%(first_name)s %(last_name)s you are late for %(location)s!" % dictionary
  • 'James Kirk you are late for The Enterprise!'

With the function locals() you can avoid going through the process of creating a dictionary. That function takes all local variables and creates a dictionary from them:

  • apples = 3
  • price = 1.85
  • total_cost = price * apples
  • "Your %(apples)d apples bought at %(price).2f cost %(total_cost).2f" % locals()
  • 'Your 3 apples bought at 1.85 cost 5.55'

For example this specifies field width (number of characters to fit the tuple in) to be 6, specifies number of characters after the decimal to be 2, and left justifies the number in the field:

  • "Pi is <%-6.2f>" % 3.14159
  • Pi is <3.14  >

Instead of using a tuple you can also use a dictionary on the right side of the modulus operator. This is helpful because you can be more specific with what values you want returned:

  • num_dict = {'e' : 2.718, 'pi' : 3.14159}
  • print("%(pi).2f - %(pi).4f - %(e).2f" % num_dict)
  • 3.14 - 3.1416 - 2.72

Any operators that work on tuples (i.e. most operators that work on lists) also work on strings.

Regular Expressions

To work with regular expressions the "re" module needs to be imported.

RegEx is a special mini language for searching strings or substrings of a particular format.

The re module has specific functions that apply RegExs to a string and return different results based on the RegEx pattern searched for.

The pattern to search for passed to the RegEx function is a string. A basic search pattern can simply be a string if you want:

  • x = "Hello, World!"
  • re.findall("Hello", x)
  • ['Hello']

There are lots of things you can put in the pattern beyond just characters in a particular order however.

Patterns can have any of three different objects in them. There are literal characters (such as the letter H in Hello above), character classes (which represent a number of different characters), and modifiers (operate on characters, character classes, or both).

Literal Characters

digits, upper/lowercase letters and punctuation characters.

Since punctuation characters are often used as modifiers there are some you have to escape with a backslash(\) to use. They are:

.^$+?*()[]|

Character Classes

These allow you to search for more than one character in a position in a pattern, where literal characters allow you to only search for one specific character.

[]

Defines a character class (searches for any characters in the brackets). This can include any letters [ahc] or a range of letters in alphabetical order [a-g].

  • x = "tisk tisk the thistle hurt my tooth"
  • re.findall("t[hi]")
  • ['ti', 'ti', 'th', 'th', 'th']

There are some escape sequences that represent common character classes. These all have equivalent [] combinations and behave the same as anything in a [], but are just shortcuts for common combinations:

Escape SequenceClass of characters
\wAlphanumerics and _
\WNon-alphanumerics
\dDigits
\DNon-digits
\sWhitespace
\SNon-whitespace

Modifiers

Can follow a character, character class or parenthesized group of characters and/or character classes. They also can expand the range of what is matched before them.

.

Searches for any character except for \n (newline)

  • x = "Hooligan owls like to hoot and hurt"
  • re.findall("h..t", x)
  • ['hoot', 'hurt']

^

Checks if the string starts with the pattern. Returns the matching pattern if the string starts with the pattern. This can only return one match because there is only one beginning to a string. As a result this is mostly useful for True/False checks. This must be at the very beginning of the pattern.

  • x = "positivity makes positivity"
  • re.findall("^p.s", x)
  • ['pos']

$

Checks if the string ends with the pattern. This must be at the very end of the pattern.

  • x = "positivity makes positivity"
  • re.findall("i.y$", x)
  • ['ity']

+

Does a check with the character before it. It sees if there is one or more occurrences of that character after the rest of the pattern being seached for.

  • x = "lee is low and likes to loot"
  • re.findall("lo+", x)
  • ['lo', 'loo']

*

+ (see above) except optional. Even if there are 0 occurrences of the character this is applied to, the rest of the pattern will be returned. Like + above it will return all occurrences of the character in a row or by itself if it is present.

  • x = "lee is low and likes to loot"
  • re.findall("lo*", x)
  • ['l', 'lo', 'l', 'loo']

{}

Like + and * except you put a number in the bracket that specifies the number of times the character before it should appear in a row.

  • x = "goofy goober gophers gallop through gorges"
  • re.findall("go{2}", x)
  • ['goo', 'goo']

You can put {#,} to mean # or more matches or {#,#} to mean between # and # number of matches.

|

Works like the or conditional. Allows you to check for different patterns at the same time.

  • x = "aiden always alliterates"
  • re.findall("[Aa]iden|always", x)
  • ['aiden', 'always']

()

Only the part of a pattern in parenthesis will be returned in a search, but RegEx outside the parenthesis will still be used in the search. This is called tagging.

  • x = "aay, bby, zzy, yyy, ccy, hhy"
  • re.findall("([a-d])[a-d]y", x)
  • ['a', 'b', 'c']

You can also use parenthesis to group entities so they behave as one entity. To use parenthesis for grouping the open paren needs to be the special sequence (?:. The closing paren can be a normal )

  • import re
  • ippat = re.compile(r'(?:\d+\.){3}\d+')
  • addrtext = 'Python web site: 132.151.1.190'
  • print(ippat.findall(addrtext))
  • ['132.151.1.190']

For more information on RegEx this website is a great resource: https://www.regular-expressions.info/

If you are going to be running the same regular expression on a variety of text it's a little more efficient if you compile the expression first with the re.compile() method.

RegEx Functions

Return a list of all matches
re.findall()
re.findall(<RegEx pattern or string to find>,<String to search through>)

This will always return a list. Any tagged expressions (expressions contained in ()) will be put into tuples within the returned list. If one expression is tagged, each matching result for that expression will go into it's own one element tuple. When multiple expressions are tagged the tuple will have an element for each of the tags. One tuple element per tag, one tuple list element per match. For example:

  • tstpat = re.compile(r'(\d+) (\w+)')
  • tstpat.findall('29 blue 83 green')
  • [('29', 'blue'), ('83', 'green')]
Return a special object called a Match Object that contains information about the search
re.search()
re.search(<RegEx pattern or string to find>,<String to search through>)

This is actually pretty useful because you can get more specific results once you process the Match Object a bit.

Note that "None" will be returned instead of the Match Object if there is no match

    • print(new_text)
    • python

    Lists

    Lists are a way to store a collection of information. The information they contain can be of any type. Even lists.

    Lists are basically what Python uses as arrays, but they are called lists because they have behavior that differs from arrays in most other languages.

    To create a list you just have to create a list of types, seperate them by commas and put this between brackets:

    number_list = [1, 2, 3, 4, 5]

    You reference an element in a list like this:

    number_list[0]

    This will return the first element of a list, as the index starts counting at 0. Since this is a bit counterintuitive it can be helpful to think of indexes as selecting the spaces between list elements.

    You can create an empty list:

    empty_list = []

    Elements of a list do not have to be the same type. This is one thing that makes lists differ from arrays in other languages:

    mixlist = [1, "dog", [10,20,30], abs]

    Elements of a list behave exactly like their individual objects behave. So for example to access a list in a list:

    • list = [1, 2, 3, [7, 10, 19, 12]]
    • list[3][2]
    • 10

    You can also use a function stored in a list like this:

    • num_func_list = [-12, abs]
    • num_func_list[1](num_func_list[0])
    • 12

    Referencing list elements

    Elements of a list are referenced with [index] at the end of the list variable like this: some_variable[some_index]

    More than just the index can be used in the reference and there are different types of notation that are used depending on how the reference is being made.

    Index notation

    To get data out of a list you can put stuff into [] after the list

    To get one element in a list use x[element index]. Remember the count of items begins at 0. Keep in mind you're selecting elements by the spaces between them. So the 0th space selects the first element:

    • nums = [1, 2, 3, 4, 5]
    • nums[0]
    • 1

    And the 1st space selects the second element:

    • nums = [1, 2, 3, 4, 5]
    • nums[1]
    • 2

    You can also get elements from the end of the list by using negatives in []. -1 indicates the last element. This is easier to visualize if you keep in mind you're selecting the space right before the last element.

    • nums = [1, 2, 3, 4, 5]
    • nums[-1]
    • 5

    -0 is equal to 0 so will select the first element:

    • nums = [1, 2, 3, 4, 5]
    • nums[-0]
    • 1

    Slice notation

    You can put more complex stuff in [] like : which indicates a slice in the list. These can be easier to understand if you keep in mind the idea indexes are selecting the spaces between elements.

    Slice a string with[<left most character index inclusive>:<right most index exclusive>]

    • nums = [1, 2, 3, 4, 5]
    • nums[2:4]
    • [3, 4]

    You can tell the slice to go to the beginning or end of the string by only putting in one number on a side of :

    So to select from the 0th space to the 3rd you can do this:

    • characters = ['A', 'B', 'C', 'D', 'E']
    • characters[:3]
    • ['A', 'B', 'C']

    And to select from the 3rd space to the end you can do this:

    • characters = ['A', 'B', 'C', 'D', 'E']
    • characters[3:]
    • ['D', 'E']

    You can also use negative numbers in slice notation. To remove the last element of a list for example:

    • characters = ['A', 'B', 'C', 'D', 'E']
    • characters[:-1]
    • ['A', 'B', 'C', 'D']

    You can select an entire list with slice notation. Just don't put anything on either side of the colon:

    • characters = ['A', 'B', 'C', 'D', 'E']
    • characters[:]
    • ['A', 'B', 'C', 'D', 'E']

    This is how you create a real copy of a list instead of just a reference. See the "Copying a list" section below.

    To append to the end of a list with slice notation whatever its length you can do this:

    • characters = ['A', 'B', 'C', 'D', 'E']
    • characters[len(characters):] = ['F', 'G']
    • print(characters)
    • ['A', 'B', 'C', 'D', 'E', 'F', 'G']

    And to append to the front of a list you can do this:

    • characters = ['A', 'B', 'C', 'D', 'E', 'F', 'G']
    • characters[:0] = ['The', 'notes', 'are:']
    • print(characters)
    • ['The', 'notes', 'are:', 'A', 'B', 'C', 'D', 'E', 'F', 'G']

    You can just get the first and last elements of a list by doing this:

    • characters = ['A', 'B', 'C', 'D', 'E']
    • characters[1:-1] = []
    • print(characters)
    • ['A', 'E']

    There are about a million more things that can be done with slice notation, but those are some examples I thought were neato.

    Modifying lists

    Index and Slice notation can both be used to modify a list, but these examples cannot be with strings as they are immutable.

    To add one element to the list:

    • x = [1, 2, 3, 4]
    • x[2] = 'tree'
    • print(x)
    • [1, 2, 'tree', 4]

    To add multiple elements to a list requires slice notation used like this:

    x[<index to add an element>:<same index on the left>]

    This will force all the elements after this index to move up one index as this does not replace anything in the list. It just adds.

    • x = ["a", "b", "c", "Baby you and me!"]
    • x[3:3] = [1, 2, 3]
    • print(x)
    • ['a', 'b', 'c', 1, 2, 3, 'Baby you and me!']

    Another way to think of this is that you are selecting the space between index 1 and 2 because if you want to select index 2 you have to use 3 on the right side of the slice. This is because you define the slice you're selecting exclusively on the right side.

    Using slice notation tells python to add the elements of a list to the list and not the whole list. If you don't use slice notation this happens

    • x = ['apple', 'banana', 'pear']
    • x[0] = ['orange', 'kiwi', 'strawberry']
    • print(x)
    • [['orange', 'kiwi', 'strawberry'], 'banana', 'pear']

    The individual elements have not been added. Instead the entire list has been added

    You can use slice notation to replace elements in a list by using the form

    x[<index to replace>:<index to replace plus 1>]

    For example:

    • x = ["mary","carrie","larry"]
    • x[1:2] = ["John","Henry","Perkins"]
    • print(x)
    • ['mary', 'John', 'Henry', 'Perkins', 'larry']

    The name at index 2 is not affected because the second index is exclusive in slice notation.

    You can use the 'del' operator to remove something from a list (see below), but slice notation works as well. Just assign a slice an empty list:

    • words = ['keep', 'keepeth', 'delete', "don't delete"]
    • words[2:3] = []
    • print(words)
    • ['keep', 'keepeth', "don't delete"]

    Keep in mind you must have a difference of at least 1 between the left and right sides of the colon when deleting a value. Otherwise you're just selecting the space between an index and the index before it. For example this doesn't remove anything:

    • words = ['keep', 'keepeth', 'delete', "don't delete"]
    • words[2:2] = []
    • print(words)
    • ['keep', 'keepeth', 'delete', "don't delete"]

    Modifying variables containing sublists

    There is some special behavior when it comes to storing variables that hold lists in a list.

    Here is an example of a variable in a list holding a sublist:

    • village_coord = [10, 15]
    • poi = [village_coord, [4, 8], [7, 6]]
    • print(poi)
    • [[10, 15], [4, 8], [7, 6]]

    There are two ways you can change the values in the sublist of poi. You can modify the village_coord list with slice or index notation:

    • village_coord = [10, 15]
    • poi = [village_coord, [4, 8], [7, 6]]
    • village_coord[0] = 5
    • print(poi)
    • [[5, 15], [4, 8], [7, 6]]

    Or you can access the sublist via poi:

    • village_coord = [10, 15]
    • poi = [village_coord, [4, 8], [7, 6]]
    • poi[0][0] = 20
    • print(village_coord)
    • print(poi)
    • [20, 15]
    • [[20, 15], [4, 8], [7, 6]]

    Notice although you are not directly interacting with the village_coord variable it is changed when interacting with the list it represents in poi.

    If, however, you change the village_coord variable by reassigning the list it represents without index or slice notation it will become disconnected from poi:

    • village_coord = [10, 15]
    • poi = [village_coord, [4, 8], [7, 6]]
    • village_coord = [99, 99]
    • print(poi)
    • [[10, 15], [4, 8], [7, 6]]

    You'll notice assigning village_coord to a new list did not change the sublist that is represented by village_coord in poi.

    To prevent the village_coord variable from being cut off from the village_coord sublist stored in the poi list you MUST modify the village_coord list with index or slice notation like this:

    • village_coord= [10, 15]
    • poi = [village_coord, [4, 8], [7, 6]]
    • village_coord[:] = [11, 16]
    • [[11, 16], [4, 8], [7, 6]]

    Copying lists

    When you assign a variable to a variable that holds a list, this creates a reference to the original list. Not a new instance of the list.

    This means any modifications you make to the new variable will also modify the original variable and its contained list:

    • one = [1, 2, 3, 4]
    • two = one
    • two.append(5)
    • print(one)
    • [1, 2, 3, 4, 5]

    To create a copy instead of a reference you use slice notation to select the entire list and assign it to a variable:

    • one = [1, 2, 3, 4]
    • two = one[:]
    • two.append(5)
    • print(one)
    • print(two)
    • [1, 2, 3, 4]
    • [1, 2, 3, 4, 5]

    This can get even more confusing when you put variables that represent lists into lists and then try to copy them. If you create a copy of a list with slice notation you will get a new list, but variables that contain lists will stay connected to both lists:

    • car_car = ['Volvo', 'Toyota', 'Saturn']
    • train = [car_car, 'boxcar', 'coalhopper']
    • train_copy = train[:]
    • train_copy[1] = 'grainhopper'
    • print(train)
    • print(train_copy)
    • [['Volvo', 'Toyota', 'Saturn'], 'boxcar', 'coalhopper']
    • [['Volvo', 'Toyota', 'Saturn'], 'grainhopper', 'coalhopper']

    As you can see these lists are now seperated. This, however, is known only as a shallow copy because the car_car variable is still shared between the two lists:

    • car_car[0] = 'Honda'
    • print(train)
    • print(train_copy)
    • [['Honda', 'Toyota', 'Saturn'], 'boxcar', 'coalhopper']
    • [['Honda', 'Toyota', 'Saturn'], 'grainhopper', 'coalhopper']

    Notice car_car[0] has changed in both lists.

    This is probably the behavior you want. It makes sense when one variable is in more than one place making a change to it anywhere should affect all instances of it. If you don't want this to happen, however, you can create what is known as a deep copy rather than a shallow copy. This requires the copy module:

    • import copy
    • car_car = ['Volvo', 'Toyota', 'Saturn']
    • train = [car_car, 'boxcar', 'coalhopper']
    • deep_train_copy = copy.deepcopy(train)
    • car_car[0] = 'BMW'
    • print(train)
    • print(deep_train_copy)
    • [['BMW', 'Toyota', 'Saturn'], 'boxcar', 'coalhopper']
    • [['Volvo', 'Toyota', 'Saturn'], 'boxcar', 'coalhopper']

    train and its sublist car_car are now completely seperated from the copy deep_train_copy.

    • names = ["Derrickson", "Clomberstank", "Albatrosm", "BoJinugs"]
    • names.sort()
    • print(names)
    • ['Albatrosm', 'BoJinugs', 'Clomberstank', 'Derrickson']

    Numerically with a list of numbers:

    • nums = [12, 13, 84, 19, 47, 30, 54]
    • nums.sort()
    • print(nums)
    • [12, 13, 19, 30, 47, 54, 84]

    Note that this sorts the list. It does not return a sorted version of the list. Therefore if you try to set results of a sort like this new_list = old_list.sort() you'll be assigning NoneType to new_list.

    You can set the reverse parameter, which defaults to False, if you would like to reverse the result of the sort method like so:

    • nums = [12, 13, 84, 19, 47, 30, 54]
    • nums.sort(reverse=True)
    • print(nums)
    • [84, 54, 47, 30, 19, 13, 12]

    The default sorting behavior requires all elements of a list to be of comparable types or else an error will occur. Strings and numbers are not comparable for example:

    • alphanumerics = [1, 'a', 2, 'b', 3, 'c']
    • alphanumerics.sort()

    • names.sort()
    • print(names)

    • print(names)
    • ['angus', 'Huebie', 'ronnie', 'Zazzmus']

    To sort like this you must return a value that you would like to sort by.

    Each element of the list is passed to the key function, modified, then passed to a temporary list. Everything in that temp list is then sorted, paired with the original value stored in the original list and then the original list is sorted to match the order in the temp list.

    This is why the parameter is a "key". This is making a temporary dictionary. By using key=<some function> you're saying this value in the list has a key equal to the result of the function. The sort method then sorts by the key side of the dictionary and saves the value side over the list the sort method is run on.

    If the key function doesn't return anything you will cause an error because the sort method will try to sort based on nothing, so it MUST return something.

    Tuples

    Basically lists that cannot be changed. They are slightly more efficient than standard lists because of this. They are required for certain functions and operations which is why they're important to understand.

    They are needed for formatting processes that use the % operator.

    Tuples can be used as keys in dictionaries

    Tuples are created like lists except they use parentheses () instead of brackets [] to be defined:

    tup = (1, 2, 3, 4, 5)

    Similarly to lists you can create an empty tuple:

    tup = ()

    Parentheses are also used for grouping in math operations so when creating a tuple with a single element a comma needs to be added:

    tup = (6,)

    If this isn't done you'll create a variable that is of the type in the parentheses and not a tuple.

    If you do this with an int you'll throw an error when attempting to reference it because you'll have created an integer that has no index reference:

    • tup = (5)
    • print(tup[0])
    • TypeError: 'int' object is not subscriptable

    If you do this with a string you'll be able to reference it with an index because strings can be referenced as a list of characters, but you won't have a tuple, you'll have a string. If you wanted to make a tuple, however, you'll get behavior you probably didn't expect:

    • tup = ('Andy')
    • print(tup[0])
    • A

    To make the string function as a single element in the tuple you must include the comma:

    • tup = ("Andy",)
    • print(tup[0])
    • 'Andy'

    Tuples value's cannot be modified without recreation because they are immutable. As a result there is no .append() method like there is with lists. Instead you have to reassign a new value to itself like this:

    • tup = (1, 2, 3)
    • tup = tup + (4,)
    • print(tup)
    • (1, 2, 3, 4)

    You can do operations that only work on lists by converting a tuple to a list and back:

    • tupl = (3,7,12,13,19)
    • ltupl = list(tupl)
    • ltupl.append(21)
    • tupl = tuple(ltupl)
    • print(tupl)
    • (3, 7, 12, 13, 19, 21)

    Even though tuples are immutable you can change mutable objects within them if that object has a variable reference outside the tuple:

    • molten_rocks = ['obsidian', 'granite']
    • rocks = (molten_rocks, 'shale', 'shist', 'othoclase')
    • molten_rocks[0] = 'marble'
    • rocks[1] = 'slate'
    • TypeError: 'tuple' object does not support item assignment

    Any immutable object can be used as a dictionary key. That includes tuples:

    • cave_location = (10, 15)
    • house_location = (17, 18)
    • pois = { cave_location : "A Cave", house_location : "A House" }
    • print(pois[10, 15])
    • 'A Cave'

    but a tuple of immutable objects cannot be used as a key:

    • sixth_floor = (['Mike', 'Mary'], ['Aiden', 'Katelyn'])
    • first_floor = ('Steve', 'Shelly')
    • residents = {sixth_floor : 'Nicest Neighbors', first_floor : 'Basement Folk'}
    • TypeError: unhashable type: 'list'

    Dictionaries

    Dictionaries are collections of elements, but they can be indexed by immutable objects like integers AND strings or tuples instead of only by consecutive numbers

    List elements are always referenced by their position in the list via index number, but people aren't great at memorizing numbers so it often makes sense to reference elements in a list via a string instead. Dictionaries allow you to do that.

    A dictionary is essentially a list of pairs. The two parts of a dictionary entry are the key (the string or other object used as an index reference) and the value stored at that index that can be refered to via the key.

    Consider how this is useful for something like a phone book. Instead of having a list of pairs of names and phone numbers and searching the list for a name then doing some work to get the number, you can simply use a name as the index for a phone number.

    Note that these are called associative arrays in other languages.

    A dictionary is created like this:

    dictionary = { <some key> : <some value>, <some key> : <some value> }

    To access a value use the key as an index just like how you would use a number index as an index for a list:

    • phonebook = {'Harvey' : '123-456-7890', 'Gary' : '420-420-6969'}
    • print(phonebook['Gary'])
    • 420-420-6969

    Adding values is even simpler than adding values to lists. Just create a new key that doesn't already exist in the dictionary and give it an assignment:

    • phonebook = {'Harvey' : '123-456-7890', 'Gary' : '420-420-6969'}
    • phonebook['Susie'] = '849-777-6969'
    • print(phonebook)
    • {'Harvey': '123-456-7890', 'Gary': '420-420-6969', 'Susie': '849-777-6969'}

    Remember that you can use any immutable object as a key. Here a dictionary is created with tuples as keys:

    • mine_positions = { (5, 4) : 'small mine', (2, 7) : 'poison mine', (8, 3) : 'big mine' }
    • print(mine_positions[(5,4)])
    • 'small mine'

    Doing this with a list instead of a tuple will result in an error:

    • pit_positions = { [5, 4] : 'deep pit' }
    • TypeError: unhashable type: 'list'

    You can reference a tuple with math problems as long as the problems solve to equal the values in the tuple:

    • mine_positions = { (5, 4) : 'small mine', (2, 7) : 'poison mine', (8, 3) : 'big mine' }
    • mine_positions[(2 * 4, 9/3)]
    • 'big mine'

    Sets

    Sets are collections of objects where membership and uniqueness is important, but actual position is not. They're lists that can only hold immutable and hashable objects (ints, strings, floats, tuples), but don't hold them in any particular order.

    Sets can use general collection operators like in, len, and be iterated over in for loops.

    To create a set use the function set() on any sequence:

    x = set(['A', 'B', 'C'])

    When you do this any duplicates will be removed. Remember sets care about uniqueness so anything that isn't unique will have copies deleted until there is only one:

    • x = set([1, 2, 3, 4, 1, 4, 1])
    • print(x)
    • {1, 2, 3, 4}

    Sets are mutable and are not hashable which means they cannot be a part of other sets. If you want a set to be part of a set, however, you can create a frozen set:

    • house_cats = frozenset(['Sable', 'Paprika', 'Juniper'])
    • all_house_cats = set(['Zipper', 'Ella', 'Frank', house_cats])
    • print(all_house_cats)
    • {frozenset({'Juniper', 'Sable', 'Paprika'}), 'Frank', 'Zipper', 'Ella'}

    Object Oriented Features

    OOP (object oriented programming) says that you want to focus on your data. An object holds all of a certain kind of data in one place.

    You should also have the functions you would like to use to modify that data with the data. These functions, when they are part of an object (i.e with the data), are called methods.

    Methods are run differently from functions. The method must be preceded by an object and the . operator:

    <some_object>.<some_method>

    For example:

    cd_names.print()

    Any additional arguments are passed the same way they are passed to functions:

    icon_for_hire.print_album_songs("Scripted")

    The Struct Module

    It is easiest to store data for your programs in string format, but when working with files generated by or used by other programs you'll likely need to work with binary data. This is what the struct module is for.

    Using 'rb' mode for the open() takes in a raw byte object, but writing code to break it into variables is a lot of work. The struct module is designed to do some of that work for you.

    Struct works a lot like RegEx, but it's specifically used for decoding byte data instead of strings.

    Here is an example:

    • import struct
    • record_format = 'hd4s'
    • record_size = struct.calcsize(record_format)
    • result_list = []
    • input = open("data", "rb")
    • while 1:
    •   record = input.read(record_size)
    •   if record = '':
    •     input.close()
    •     break
    •   result_list.append(struct.unpack(record_format, record))

    record_format = 'hd4s' is like a RegEx pattern. It means a single short C integer (h), a single double-precise C float (d), a string 4 characters long, (4s). This pattern is going to be applied to a list of records written in C.

    This pattern is going to be applied to a series of bytes. But each time the pattern is used you'll only want to use it on a record the size of the pattern. Anything outside of the pattern match is not valuable information so you only want to use this pattern on a length of bytes equal to the length of bytes of the pattern. This is what struct.calcsize(record_format) calculates. It finds the length of the pattern in bytes.

    record = input.read(record_size)Remember the read() method takes an optional "bytes to read" when it runs. Giving it record_size means it will read the bytes of the file in chunks of bytes equal to the byte size of the pattern. Without this it would read the entire file as one long string of bytes representing the entire file. Each time the read function runs on the same file it will continue where the last read function ran. So when this is in a loop it will read the entire file, but only in chunks equal to record_size

    Finally each chunk of bytes returned by input.read(record_size) has the translation pattern of record_format = 'hd4s' applied to it with struct here: result_list.append(struct.unpack(record_format, record)) Note the method struct.unpack() returns a tuple of python values.

    The Struct module also can pack stuff into byte objects with struct.pack(). This takes a format to pack things into and values to pack up:

    • import struct
    • record_format = 'hd4s'
    • struct.pack(record_format, 7, 3.14, 'gbye')
    • b'\x07\x00\x00\x00\x1f\x85\xebQ\xb8\x1e\t@gbye'

    You can also insert special characters that describe in more detail the format the data should be written in if you choose.

    Python has the unique ability to write any data structure into a file or read a data structure out of a file and re-create it. This means you can save any variable, object, or state of your program and retrieve it.

    This is done with the pickle module:

    • import pickle
    • a = <some object or variable or whatever>
    • b = <another object or variable or whatever>
    • c = <even more of an object or variable or whatever>
    • file = open("Program State", 'w')
    • pickle.dump(a, file)
    • pickle.dump(b, file)
    • pickle.dump(c, file)
    • file.close()

    The purpose of pickling is that you can store anything in a, b or c without worrying about decoding or encoding it in any way. To get it back you simply do this:

    • import pickle
    • file = open("Program State", 'r')
    • a = pickle.load(file)
    • b = pickle.load(file)
    • c = pickle.load(file)
    • file.close

    Opens a file
    open()
    open(<filename as string>, <mode as string>, <newline=>)
    <filename as string>

    This is the pathname of the file to be written or modified, passed as a string of course.

    <mode as string>

    You'll want to set the mode based on what you want to do with the file. The options you'll pass as a string are these:

    Argumentmeaning
    'r'Open an existing file for reading (This is the default)
    'rb'Binary Mode: Open a non-text file. Used to get raw data.
    'w'Open file for writing. Creates new file if there isn't one
    'a'Open file for appending. Creates new file if there isn't one
    'r+'Open file for reading/writing. If file exists content isn't touched
    'w+'Open file for reading/writing. Content overwritten for existing file
    'a+'Essentially 'r+'

    The default is 'r'. Note that r does some translating of text. Operating systems don't necessarily use \n to mean 'newline', but it would be a pain to have to use a different read mode for every os. So 'r' translates all newlines into the python newline escape character '\n'

    Binary Mode ('rb') is useful when working with raw non-text data because it doesn't bother doing the translations of text that 'r' does. If this happened with non-string info it might corrupt the data. Anything read in this mode will return a bytes object, not a string. This will have to be decoded into a string if you want it in that form. This is important with network protocals as that info will likely by in byte form.

    <newline=>

    Specifies the treatment of newline characters. For example newline='\n' means only '\n' will be treated as a newline character.

    This does not read the file. Instead this creates a file object that you can use methods on that allows you to interact with a file. For example:

    • file_object = open('myfile', 'r')
    • first_line = file_object.readline()

    There is a third argument set by default. This argument is usually not needed, but tells Python how to buffer the file.

    0 means no buffering. Anything written to the file immediately appears in it.

    1 means line buffering. Info is only written after a newline character is sent to the file.

    Any other value is the size of the buffer used when transferring information to the file.

    Reads one line from the file
    x.readline()

    This method reads one line from the file. Each subsequent time it runs it reads the next line of the file until every line is read.

    Every line returned by this will have a newline character at the end of it so you'll probably want to remove that.

    To read the first line of a file:

    • f = open('inputfile','r')
    • line = f.readline()
    • line = line[:-1]

    To keep running this through an entire file a while loop can be used:

    • f = open('inputfile','r')
    • while True:
    •  line = f.readline()
    •  if not line:
    •   break
    •  print(line)
    • f.close()
    Reads an entire file and returns a list of elements representing each line of the file.
    x.readlines()

    Once a call is made, subsequent calls will return an empty list unless the file is rewound or closed.

    Reads entire file as a single scalar string
    x.read()
    <file object>.read(<Bytes to read>)

    This basically encodes everything into one long string.

    You can give this an optional integer that is the number of bytes read. When that argument is supplied subsequent calls to read will continue the read from where the previous call stopped.

    • input_file = open("myfile", 'rb')
    • header = input_file.read(4)
    • data = input_file.read()
    • input_file.close()

    Note that the file is opened in binary mode. This is to prevent 'r' from translating newlines in the file. Usually you use the read() method when you just want raw data, but if the data is being changed in ANY way it can be corrupted.

    Write a single scalar object to the file object file
    x.write()
    <file object>.write(<something to write to file>)

    This method does not append a newline to the end of each line. This lets it be used for all types of files, not just text files. It will, however, translate any newline characters as a new line.

    Remember the file object must have been declared with an argument indicating it can be written to for this to work.

    Writes a list
    x.writelines()
    <file object>.writelines(<list of items to write>)

    This does not append a newline to the end of each line. It is simply the version of write() that takes a list. It concatenates them together and will only put strings on newlines if they end with a newline character.

    This is the inverse of readlines. If this runs on a list returned by readlines() it will create a copy of the file read by it.

    Remember the file object must have been declared with an argument indicating it can be written to for this to work.

    Close a file

    Python automatically closes files when the program terminates or you attempt to re-open a file. When working with multiple files in one program, however, it is a good idea to close what isn't being used. This frees up system memory and allows the file to be worked with by other programs or functions.

    Empty a file's buffer

    This is useful because you can ensure a file's buffers are empty. If an argument passed to the write function isn't showing up in the file you may have to flush the buffer.

    Set the location in the file the next read or write operation will occur.
    x.seek()
    <some file>.(<offset in bytes>,<where offset starts from>)

    Using the read/write functions go through the file in order. This lets you change where you are in the file.

    To figure out the offset in bytes you can use the tell method below.

    The parameter for where the offset starts can have three values: 0, 1, or 2.

    0 mean the offset is made relative to the beginning of the file. 1 means it's made at the current position in the file, and 2 means the end of the file.

    This function is very powerful because you can "rewind" back to the beginning of the file by doing file.seek(0)

    Get the current offset of a file in bytes
    x.tell()

    This returns the location in bytes of where the next write or read will take place in the file.

    Change file length based on bytes
    x.truncate()

    Takes a single integer argument that is the bytes long the file should be. If passed a value longer than the current file the extra space will be filled with zeros.

    Indentation

    Python uses indentation to define blocks of statements, rather than the {} that other languages tend to use.

    When defining a block there is no fixed number of indents that you need to use. The entire block just needs to be on the same indentation level.

    Since this behavior is so different from other languages it's important to keep in mind this can cause issues when working on the same file with two different editors. Sometimes tabs can be converted to spaces or vice versa when switching editors. This is more likely to cause issues if you're using spaces and tabs to do your indents in one program. Stick to one or the other.

    There are two main reasons Python integrates indentation into its syntax. The first is it makes the structure of the code easier to understand at a glance. Languages that use brackets have more flexibility, but this means each coder's subjective use of them will cause the program structure to look different. In Python the structure is made clear through the syntactical rules. This similarity of code between coders leads into the next benefit: Code becomes easier for everyone to read if it must look the same for everyone.

    Indented blocks are proceeded by compound statements. These are lines of control flow code such as if, elif, else, while, and for loops.

    In general statements are placed on new lines, but you can use a semicolon to effectively end a line and start a new one virtually:

    • x = 7; y = 8; print(x + y)
    • 15

    You can do the same thing with a colon, but only if it's the colon of a compound statement:

    • x = 7
    • y = 8
    • if y > x: print("y is bigger")
    • y is bigger

    You can break up single lines of code with a \. The indentation level on the second line does not matter:

    • print(7 + \
    • 8)
    • 15

    You can also breakup lines anything within {}, [] or () when you are declaring a dictionary, list, or tuple. Again indentation level on the second line doesn't matter:

    • the_biggest_list_of_all_time = ['An entire sentence.', 'Another full sentence',
    •           'Yet another sentence.', 'The final sentence.']
    • print(the_biggest_list_of_all_time)
    • ['An entire sentence.', 'Another full sentence', 'Yet another sentence.', 'The final sentence.']

    To have a string cross to the next line you have to end the string and put a backslash after it. Python is smart though and automatically concats any whitespace characters between strings so although you have to open quotes on a new line it will function as one line:

    • print("Okay so you have a really really long string and you want to be " \
    •           "able to see it all in your window.")
    • Okay so you have a really really long string and you want to be able to see it all in your window.

    Conditional Execution (If statements)

    This gives you the ability to execute groups of statements only if some logical condition is true. It is done through operators like the if statement.

    if

    Checks a conditional and runs the block of code beneath it if it is True:

    • I = "Master Coder"
    • if I == "Master Coder":
    •   print("Good Work!")
    • "Good Work!"

    else

    The block of code under else needs an if statement to run. If the if statement above else is false then the code under else will execute:

    • I = "Okay Coder"
    • if I == "Master Coder":
    •   print("Good Work!")
    • else:
    •   print("Keep Working at it!")
    • Keep Working at it!

    You'll notice else doesn't have a logical expression in it. It only runs if the logical expression in the if statement above it is false. You can create an else statement that checks for a logical expression with the elif operator.

    elif

    An else statement with an included logical expression. This needs to have an if statement above the block of code above it. Again, this and the entire block of code beneath it, will be skipped if the if statement's logical expression is true.     • if I == "Master Coder":
    •   print("Good Work!")

    •   print("You're doing great!")
    • else:
    • print("Wherever you are, you'll get there!")
    • You're doing great!

    Keep in mind if and elif statements are evaluated until one is found to be true. This means in situations where only one of several statements can be true, they should all be part of an if/elif chain. This is more efficient then a bunch of if statements because any blocks after a true statement will be skipped.

    For loop

    This lets you run a block of code for every element in an iterable (lists, tuples, strings, and even some functions) and also lets you interact with every element the loop runs for.

    • nums = [1, 2, 3, 4, 5]
    • for i in nums:
    •   print(i)
    • 1
    • 2
    • 3
    • 4
    • 5

    The variable "i" can be anything. This is called the For Loop Variable.

    For loops can let you "unpack" items out of a list in a list. You just have to use index notation on the variable declared in the for loop that represents each list in the list:

    • names = [['Mark', 'Andrews'], ['David', 'Bowie'], ['Cody', 'Tobrias']]
    • for full_name in names:
    •   print(full_name[0] + ' ' + full_name[1])
    • Mark Andrews
    • David Bowie
    • Cody Tobrias

    For better readability you can do the unpacking when the for loop gets declared by using multiple for loop variables:

    • names = [['Mark', 'Andrews'], ['David', 'Bowie'], ['Cody', 'Tobrias']]
    • for first_name, last_name in names:
    •   print(first_name + ' ' + last_name)
    • Mark Andrews
    • David Bowie
    • Cody Tobrias

    The for loop variable only represents a copy of elements in a sequence. As a result you cannot use a for loop to directly change values in a list:

    • nums = [1, 2, 3, 4, 5]
    • for num in nums:
    •   num = 10
    • print(nums)
    • [1, 2, 3, 4, 5]

    One solution to this is to create a new list and appending all the changed variables of the original list to it:

    • nums = [1, 2, 3, 4, 5]
    • nums_modded = []
    • for num in nums:
    •   nums_modded.append(num + 1)
    • print(nums_modded)
    • [2, 3, 4, 5, 6]

    Something else to keep in mind is that each element of the list can be modified with index notation:

    • nums = [1, 2, 3, 4, 5]
    • for num in nums:
    •   nums[num - 1] = "Num"
    • print(nums)
    • ['Num', 'Num', 'Num', 'Num', 'Num']

    It's unlikely the data stored in the sequence the for loop runs for will only be -1 away from the correct index of each item in the list. You can do a trick with the range() function to sync up the times the for loop has ran with the index of each element in the sequence passed to the for loop:

    • colors = ['red', 'blue', 'green', 'orange', 'rose']
    • for i in range(len(colors)):
    •   if colors[i][0] == 'r':
    •     colors[i] = 'reddish'
    • print(colors)
    • ['reddish', 'blue', 'green', 'orange', 'reddish']

    List comprehensions

    A very common way to use a for loop is to declare a new list, run the loop on an old list, and append the result of each iteration of the for loop to the new list:

    • nums = [1, 2, 3, 4, 5]
    • doubled_nums = []
    • for num in nums:
    •   doubled_nums.append(num * 2)
    • print(doubled_nums)
    • [2, 4, 6, 8, 10]

    This use of for loops is so common Python has created a shortcut called a list comprehension. This allows you to assign the for loop's results directly to the new list you normally have to make before running the for loop and append values to.

    The basic format a list comprehension follows is new_list = [<some expression> for <for loop variable> in <some original list> if <logical expression (optional)>]

    So the code above can be done like this:

    • nums = [1, 2, 3, 4, 5]
    • double_nums = [num * 2 for num in nums]
    • print(double_nums)
    • [2, 4, 6, 8, 10]

    and with an if statement:

    • nums = [1, 2, 3, 4, 5, 6, 7, 8]
    • double_nums = [num * 2 for num in nums if num % 2 == 0]
    • print(double_nums)
    • [4, 8, 12, 16]

    You can also use them with dictionaries, but you need to supply variables for key : value pairs and put the comprehension in {} instead of []:

    • x = [1, 2, 3, 4]
    • x_squared_dict = {item: item * item for item in x}
    • print(x_squared_dict)
    • {1: 1, 2: 4, 3: 9, 4: 16}

    A list comprehension can contain multiple for statements and multiple sequences. The equivalent of running a for loop for each variable in a sequence for example:

    • first_letters = ['A', 'B', 'C']
    • second_letters = ['A', 'B', 'C']
    • [a + b for a in first_letters for b in second_letters]
    • ['AA', 'AB', 'AC', 'BA', 'BB', 'BC', 'CA', 'CB', 'CC']

    Installing other Modules

    A module is simply a file containing code. It defines a group of Python functions or other objects. They mostly contain Python source code, but some contain compiled C or C++ object files.

    Modules are installed using the import statement at the top of a file like import shutil. "shutil" in this case is the name of a module. You can import more than one module by placing commas between module names.

    Importing places all the module's variables and methods into the namespace of the program. If you don't want the entire module in the namespace you can specify only the modules or variables you want access to by placing a "." and the name of the method/variable after the module name in the import line.

    When importing an entire module you must reference methods of the module like so: <module>.<method> so to use the move method of shutil: shutil.move()

    If you only want the move method of the shutile module you can import just that with this alternative import statement: from shutile import move if this is the way move is imported you don't need the module name when referencing this method of it. Instead you can call it just with move().

    You can import all the methods of a module so that you don't have to specify which module you're referencing by doing this: from shutile import * this is a bit risky, however, because methods imported this way will overwrite any methods that share names. This is only useful if the writer of the module specifically made the module to be imported this way and made sure none of the method names overwrite any default methods.

    In general it makes sense to import a module and reference it in the <module name>.<method name> format. This makes it clearer what you're doing throughout the program.

    Modules help avoid name-clash problems. If you write a module that has a function reverse() there can be other modules with the same common function name reverse(). This is because of the Python use of namespaces which is a dictionary of identifiers available to a block, function, class, module, etc.

    Here is an example of os.walk used in a script to calculate the size of all files in the current directory as well as those in its subdirectories:

    • import os
    • total = 0
    • for root, dirs, files in os.walk('.'):
    •   size = sum(os.path.getsize(os.path.join(root, file)) for file in files)
    •   total += size
    • print("Total Size of all Files:", total)

    os.path.getsize returns the size of a given file (note that we need to pass the entire file path, not just the filename). The file path is constructed with os.path.join. The first argument to os.path.join is the root directory, followed by the filename.

    Using list comprehension, we generate a list of all the sizes of the files in the current directory. We then get the sum of all the sizes with the sum function.

    As we iterate through each subdirectory, we get a new list of files. We keep track of the total with the total variable.

    Makes a copy of a single file
    shutil.copyfile()

    First argument is the file to be copied and the second is the full name of the desired copy.

    Can copy a file into a specified directory
    shutil.copy()

    Same as shutil.copyfile but the second argument can be a directory to copy the file to

    Copies an entire directory tree
    shutil.copytree()

    Takes as arguments the location to copy from and the location to copy to.

    You can also give this a third argument called symlinks. When it is set to true the directory tree is made up of links. When it is set to false or not there copies of actual files are taken to the destination directory.

    Removes an entire directory tree
    shutil.rmtree()

    The only argument this takes is the directory path to be removed.
